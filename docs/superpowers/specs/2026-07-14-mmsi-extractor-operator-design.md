# MMSI 提取算子设计

> 日期：2026-07-14  
> 状态：已确认  
> 参考：`docs/mmsi-set-operations-workflow-node-design.md`

## 1. 目标与范围

在工作流“算子业务逻辑”分组中新增第 4 个独立算子“提取 MMSI 集合”。该节点从一个或多个前序对象数组中提取元素顶层的 `mmsi` 字段，规范化并稳定去重后，用英文逗号连接为字符串输出。

现有 MMSI 交集、并集、差集节点保持不变。本次不提供可编辑代码、不允许修改输出名称，也不扩展为通用字段提取节点。

## 2. 节点身份与持久化

- 新增独立的前端 `StandardNodeType` 和对齐的后端 `NodeTypeMetas.ID`，固定使用 MMSI 节点号段的下一个值 `2004`。
- 后端字符串类型使用独立 key `MirapMMSIExtractor`。
- 节点类别固定为 `operator_logic`，图标和颜色沿用现有 MMSI 逻辑算子。
- `WorkflowNodeRegistry.type` 与 `meta.nodeDTOType` 都指向新的独立类型，保证保存、复制、发布和重新打开后仍按专属节点恢复，不退化成普通代码节点。
- 前端同时注册到 `NODES_V2`、节点 registry 汇总和 enabled-node-types 映射。

## 3. 配置与契约

配置面板只展示“输入结果集”：

- 允许选择一个或多个前序输出。
- 每个输入必须是对象数组；至少配置一个输入。
- 保存时使用稳定的运行时参数名 `dataset_1`、`dataset_2`，依次对应用户选择顺序。
- `formatOnSubmit` 必须把引用表达式保存为完整的 `ValueExpressionDTO`；`formatOnInit` 必须把 DTO 恢复为表单引用。
- 输入变化或重新连线后按当前引用重建参数映射，不沿用已经错位的 `dataset_N`。

输出契约固定为：

```json
{
  "mmsis": "371836000,477123456"
}
```

- 输出变量名固定为 `mmsis`。
- 输出类型固定为字符串。
- 没有有效 MMSI 时输出空字符串。

## 4. 提取与去重规则

节点按输入选择顺序遍历数据集，并按每个数组原始记录顺序遍历元素：

1. 元素必须是 JSON 对象；其他元素跳过。
2. 只读取对象顶层 `mmsi`，不支持嵌套路径。
3. `mmsi` 只接受字符串或数字；布尔值、对象、数组、`null` 和缺失值跳过。
4. 字符串和数字统一转换为字符串，再去除首尾空白。
5. 转换后为空字符串的值跳过。
6. 使用转换后的精确字符串作为去重 key；`"00123"` 与 `"123"` 是两个不同值。
7. 保留每个 key 第一次出现的位置，后续重复值忽略。
8. 最终用英文逗号 `,` 连接，不插入额外空格。

例如：

```text
dataset_1 = [{"mmsi": 2}, {"mmsi": " 1 "}]
dataset_2 = [{"mmsi": "1"}, {"mmsi": 3}]
输出 = {"mmsis": "2,1,3"}
```

## 5. 执行架构

节点采用独立 NodeType 和专属 adaptor，但执行逻辑按封装代码节点思路实现：

- 前端不暴露代码编辑器，只保存上游输入引用和固定输出 schema。
- 后端 adaptor 将输入映射和输出类型转换为运行态 `NodeSchema`。
- Build 阶段构造专属 runner；runner 调用仓库现有 `backend/infra/coderunner` 执行内部固定的 Python 模板，不接受画布数据中的任意代码。
- 固定 Python 模板由后端代码常量提供，避免客户端篡改代码内容或历史草稿携带任意脚本。
- runner 把多个 `dataset_N` 作为参数传入代码执行器，并只接受/返回固定的 `mmsis` 字符串字段。

当前定制仓库保留了 CodeRunner 元数据和基础设施执行器，但工作流 adaptor 目录中没有注册普通 CodeRunner 节点。因此本节点复用的是基础设施 `coderunner.Runner`，不依赖一个不存在于当前节点目录中的普通代码节点 Config。

## 6. 错误处理

- 未配置输入：前端校验阻止保存/试运行，后端 adaptor 同时返回明确配置错误。
- 运行时某个 `dataset_N` 不是数组：节点执行失败并指出对应输入名；不静默将错误类型当成空数组。
- 数组内存在非对象元素或无效 `mmsi`：跳过该元素，不使整个节点失败。
- 代码执行器未初始化、超时或返回非预期结构：沿用 coderunner 错误链路，并补充节点上下文。
- 所有输入均为空数组，或均无有效 MMSI：正常返回 `{"mmsis": ""}`。

## 7. 单节点试运行与保存恢复

- `node-test.ts` 必须根据每个已选上游引用生成试运行输入，而不是仅返回字段名。
- transformer 测试必须确认多个引用保存后仍有对应的 `inputParameters`，且 `blockID`、输出字段名和 `dataset_N` 对齐。
- 保存后重新初始化表单，应恢复相同数量、顺序和来源的输入引用。
- 手工检查保存 payload 中新节点类型为独立 ID `2004`，`inputs.inputParameters` 非空。
- 手工重新打开工作流，节点仍显示为“提取 MMSI 集合”，配置和输出引用不丢失。

## 8. 测试范围

前端 Vitest 覆盖：

- 节点 registry 的独立类型、DTO 类型、分类注册和固定字符串输出。
- 一个及多个对象数组引用的提交转换和初始化恢复。
- 引用转换常规路径与 variable facade 兜底路径。
- 未选择输入时的表单校验。
- `node-test.ts` 生成多个 `dataset_N` 输入。

后端 Go 测试覆盖：

- NodeType ID、类别、可执行元数据和 adaptor 注册。
- 单输入提取与英文逗号拼接。
- 多输入按选择顺序合并并稳定去重。
- 数字和字符串规范化、首尾空白、前导零精确比较。
- 缺失值、`null`、空字符串、布尔值、对象、数组和非对象数组元素跳过。
- 空数组或全无效数据返回空字符串。
- 非数组运行时输入返回明确错误。
- coderunner 失败和异常返回结构被正确传播。

## 9. 非目标

- 不支持自定义提取字段或嵌套路径。
- 不排序 MMSI。
- 不校验 MMSI 位数或业务合法性。
- 不返回字符串数组或对象数组。
- 不支持用户修改分隔符或输出变量名。
- 不修改现有交集、并集、差集算法。
